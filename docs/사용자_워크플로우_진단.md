# 결(Gyeol) — 6개 페르소나 워크플로우 & UX 진단 (3판)

> 작성일: 2026-05-29 (3판 — 가상 데이터 제거·완전 초기화·무료 내보내기·검색 반영)
> 범위: 앱 시작(온보딩) → 기록(캡처/회고/아카이브/검색) → 결제(플랜/백업/책)
> 기준 화면 폭: 모바일 카드(≤480px)

## 테스트 방법 메모

실제 브라우저 클릭이 아니라, **현재 코드 경로를 따라간 인지적 워크스루**다. 2판 이후 적용된 변경을 반영했다.

**시작 상태 (2·3판의 핵심 변화):**
- 모든 사용자가 **완전히 빈 상태**로 시작. 개인 공간 하나, 결 0개, 플랜 없음. **가상/시드/데모 데이터는 코드에서 전면 제거**됐다(`mockFragments.ts`·`lib/demo.ts` 삭제, 데모 모드 UI 제거).
- 앱 부팅 시 **1회성 완전 초기화**(`lib/bootReset.ts`)로 이전에 캐시된 가짜 데이터·플래그·미디어까지 모두 비워진 뒤 시작한다.

**심각도:** `P0` 정체성·핵심경험 훼손 / `P1` 흐름 막힘·강한 혼란 / `P2` 다듬기.

## 1. 처음 사용하는 사용자

### 워크플로우
1. 진입 → `오늘`, **WelcomeNote** "환영해요. / 지금, 무엇이든 한 결을 가볍게 남겨주세요." (가입·튜토리얼 없음)
2. 캘린더/회고 모두 비어 있고, **캘린더엔 "아직 기록한 날이 없어요. 오늘 첫 결을 남겨보세요." 안내**(죄책감 없는 톤, 탭하면 오늘로) ✅.
3. 매체 토글/캡처 진입 → 저장 → "오늘 남긴 결 1" (내가 쓴 것만).
4. 같은 날 재진입해도 프롬프트 고정 ✅.

### 👍 잘 된 점
- **진짜 빈 페이지에서 시작.** 첫 가치 경험이 정직해짐(1판 최대 문제 해소).
- 가입 없는 첫 캡처, 4매체 동급, 오프라인 저장, 죄책감 장치 없음.
- 빈 캘린더에 따뜻한 첫-결 유도가 생겨 빈 상태가 막막하지 않음.

### ⚠️ 문제 / 어색
- WelcomeNote 이후 Day 2~3을 받쳐주는 건 프롬프트 칩. 충분하나, 복귀 후크는 여전히 가벼운 편(P2).
- 알림 미작동으로 Day 1 복귀 트리거가 없음(C8, 후속).

## 2. 1년 이상 사용 중인 사용자

### 워크플로우
1. `캘린더` → 월 이동, **"1년 전 오늘"**, 우상단 🔍 **검색**.
2. `돌아보기` → 주/월/연. 연 뷰: 대표 Hero + 매체별 합계 + 월 타일.
3. `오늘` 결 5개↑ → 주간 회고 카드 자동 노출.
4. 설정 → **내 결 내려받기(무료)**로 1년치를 파일로 보관, 또는 클라우드 백업(Plus).

### 👍 잘 된 점
- "1년 전 오늘" + 연간 회고가 강한 장기 보상.
- **검색 추가**로 수백 결에서 제목·날짜로 찾기 가능(1·2판 핵심 공백 해소).
- **무료 내보내기로 데이터 탈출구 확보**(N2 해결) — 기기 변경·캐시 삭제 대비.

### ⚠️ 문제 / 어색
- 실제 클라우드 동기화는 아직 시뮬레이션(IDB) — 기기 간 자동 동기화는 백엔드 필요(P1, 후속). 단 무료 내보내기로 수동 이전은 가능해짐.
- 연 뷰 월 타일 탭 → 그 달 "가장 최근 결" 하루로 점프(월 전체 뷰 부재)(P2).
- 검색이 본문/태그까지는 못 감(N4).

## 3. 개인 공간만 사용하는 사용자

### 워크플로우
1. 공간 칩 "내 결" 고정. 캡처 → 오늘/회고/캘린더.
2. 과거 빈 날 탭 → 백필("이 날에…") → **"시간 미상"** 표기.
3. 설정 → 내 결 내려받기(무료).

### 👍 잘 된 점
- 가장 깔끔한 경로. 무료·빈 시작이 자연스럽고 미결제 Plus 노출 없음 ✅.
- 백필이 정직(가짜 시각 대신 "시간 미상"), 무료 내보내기로 백업 안심.

### ⚠️ 문제 / 어색
- 백필 정렬 앵커가 정오라 같은 날 여러 백필이 같은 시각으로 묶임(P2, 경미).
- 솔로 사용자에게도 "새 공유 공간" 유도가 항상 보임(P2, 경미).

---

## 4. 공유 공간을 사용하는 사용자

### 워크플로우
1. 공간 칩 → "새 공유 공간" 생성(시작 시 공유 공간 없음 — 직접 만든다).
2. "이 공간에 초대하기" → 링크 복사 → `/invite/:token`(초대자 콘텐츠 선노출로 빈 방 방지 ✅).
3. 같은 날짜 페이지를 멤버별 비동기로 채움.

### 👍 잘 된 점
- 빈 방 방지·비동기 설계·떠나기/닫기 결 이동 안내가 견고.
- 공간별 색 dot, 작성자 표시 등 공동 페이지 기본기 탄탄.

### ⚠️ 문제 / 어색
- **C5 (P1, 후속):** 단일 사용자 mock이라 **실제 2인 비동기 채움을 체험할 수 없다.** (이전의 데모 주입기도 제거됨 — 진짜 검증은 백엔드/실제 초대 수락 필요.)
- 초대 링크가 `window.location.origin` 기반 — 네이티브 출시 시 딥링크 필요(P1, 후속).
- 공유 공간 멤버 상한(2~6명) UI 미강제(P2).
- 공유 공간 백업은 결제 불가라 막힘(시트 내 "곧 열려요"). 단 **무료 내보내기로 내 결은 빼낼 수 있음**.

## 5. 플랜·"책 만들기" 결제에 관심 있는 사용자

### 워크플로우 (의도 vs 현실)
1. 설정 → "결 플랜 보기" → `PlansPage`(무료 Hero → Plus·개인 → 그룹·페어 → 프린트 → 약속).
2. 사고 싶다 → 버튼 **전부 `disabled`**("곧 만나요"). 결제 진행 안 됨.

### 👍 잘 된 점
- 플랜 페이지 정보 설계·카피가 명확해 욕구를 잘 끌어올림.

### ⚠️ 문제 / 어색
- **C2 (P0, 의도된 비범위):** 아무것도 살 수 없음. 결제 의향 사용자가 전면 차단. 백엔드/결제 단계에서만 해소.
- "책 만들기" 진입 동선 약함 — 연간 회고에서의 자연 진입(§14.4 계획) 미구현(P1).
- "알림 받기" 버튼이 실제 lead 수집 안 함(P1).

## 6. 결제에 관심이 없는 사용자

### 워크플로우
1. 캡처·회고·캘린더·검색 자유 사용 — 결제 팝업·배너 없음 ✅.
2. 설정 → **내 결 내려받기(무료)**로 데이터 보관 가능 ✅. 백업하기는 무료 공간이면 시트 안에서 "곧 열려요" 안내(튕기지 않음).

### 👍 잘 된 점
- 결제 압박 거의 없음. 과거 열람·내보내기 모두 무료 — §14.2 "본인 데이터는 항상 무료" 정확히 준수.
- 무료 사용자도 **데이터 탈출구가 생겨**, "평생 무료 사용자가 갇히는" 위험(2판 N2) 해소.

### ⚠️ 문제 / 어색
- **N3 (P2):** 백업 시트가 가격·플랜 배지를 노출(약한 노이즈). "내 결 내려받기(무료)"가 위에 있어 완화됐지만, 백업/내보내기 두 표면의 구분이 처음엔 헷갈릴 수 있음.

## 우선순위 정리 (제안)

### 지금(로컬 목 단계)에서 더 다듬을 만한 것
1. **검색 강화(N4)** — 태그(후속)·매체 필터 칩(사진/영상/글/음성) 정도는 로컬에서 가능.
2. **연 뷰 → 월 전용 뷰** (월 타일 탭 시 그 달 전체).
3. **데이터 표면 정리(N3)** — "내려받기(무료)"와 "백업(Plus 클라우드)"의 관계를 한 줄로 더 분명히.
4. (선택) 백필 시각 선택 옵션, Day 2~3 복귀 후크.

### 백엔드/네이티브 단계로 미룰 것
- **C2 실제 결제(Plus·그룹·프린트).** 5번 페르소나 전환.
- **실제 클라우드 동기화·인증·진짜 초대 수락(2인 비동기, C5).** 2·4번 핵심.
- **실제 알림 발송(C8)** / "알림 받기" lead 수집 / 딥링크 초대.

### 한 줄 총평
1·2판에서 지적한 거의 모든 P0/P1(가짜 시드·미결제 Plus·백업 막다른 길·파괴적 데모 토글·데이터 탈출구 부재·검색 공백)이 **해소**됐다. 솔로 경험(1·2·3·6번)은 정직하고 자급적이며 압박이 없다. 남은 큰 덩어리는 **결제(C2)와 실제 다인 동기화(C5)** 둘뿐이고, 이는 백엔드 단계의 일이다. 로컬 단계에서 제품의 형태는 사실상 완성에 가깝다.
